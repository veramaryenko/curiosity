# Specs — plany nowych funkcji

Każdy feature PM opisuje w osobnym pliku w tym katalogu.

## Format pliku spec

```
specs/
  YYYY-MM-DD-nazwa-funkcji.md
```

## Szablon

```markdown
# Nazwa funkcji

## Co to robi (dla użytkownika)
<!-- Krótko — co zmienia się w UX -->

## Gdzie w aplikacji
<!-- Które strony/komponenty to dotyczy -->

## Kryteria akceptacji
- [ ] ...
- [ ] ...

## Nie robimy (scope)
<!-- Co celowo pomijamy w tym featurze -->
```
